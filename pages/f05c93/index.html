部署后端（flask+gunicorn+nginx）

# flask的部署

# 本地测试

这个可以采用python自己启动，方法就是写好一个flask文件之后，直接在那个地址

python app:app

第一个app是指文件名，第二个app是flask文件里面的app。

# 服务器启动

需要使用gunicorn进行守护线程，第一步是安装。

pip install gunicorn

第二步是一顿cd进入文件所在地址后，先写一个配置文件，保存为任意名字，越简单越好，我就命名为a.py。

# 绑定的ip与端口
bind = '172.17.91.250:5252'
           
# 设置超时时间120s，默认为30s。按自己的需求进行设置timeout = 120
timeout = 120      
# 超时重启
graceful_timeout = 300
# 在keep-alive连接上等待请求的秒数，默认情况下值为2。一般设定在1~5秒之间。

第三步就可以启动了，在我的实验室服务器上，启动的代码应该是。

/home/web_test/miniconda3/bin/gunicorn -c a.py app:app

# 配置nginx

location ^~ /DLpMHCIapi/{
		#DLpMHC I的后端接口跨域处理，由gunicorn守护
		proxy_pass http://39.104.48.98:5252/;
		
		#转发代理，设置请求头允许跨域
		add_header 'Access-Control-Allow-Origin' *;
		add_header 'Access-Control-Allow-Credentials' 'true';
		add_header 'Access-Control-Allow-Methods' *;
		add_header 'Access-Control-Allow-Headers' *;
}
