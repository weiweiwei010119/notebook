部署前端（vue+nginx）

# 打包并上传到服务器

首先在根目录创建一个deploy.sh

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd dist

scp -r * web_test@39.104.48.98:/home/web_test/DLp/dist/DLpMHCI


cd -

然后再去package.json里面修改成这个样子。

  "scripts": {
    "serve": "vue-cli-service serve",
    "build": "vue-cli-service build",
    "deploy": "bash deploy.sh"  #主要是加上这个
  },

这样代码写好之后，跑一下npm run deploy就o的k了。

# 配置nginx

server {

    listen       80;
	
    server_name  www.jianglab.org.cn jianglab.org.cn 39.104.48.98;
    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;
	
	location /{
		proxy_pass http://www.baidu.com;
	}
                                    
	location /DLpMHCI {
		#DLpMHC I的前端网页请求，转发至vue生产环境文件
		root /home/web_test/DLp/dist;
		index index.html index.htm;
		
		#转发代理，设置请求头允许跨域
		add_header 'Access-Control-Allow-Origin' *;
		add_header 'Access-Control-Allow-Credentials' 'true';
		add_header 'Access-Control-Allow-Methods' *;
		add_header 'Access-Control-Allow-Headers' *;
	}
}

# tip

尤其要看文件的路径，这个特别有意思 虽然在nginx里面写的root是/home/web_test/DLp/dist，但他实际上去/home/web_test/DLp/dist/DLpMHCI这个路径里面去寻找，也就是说在这个目录里面需要有一个叫做index.html的文件。
